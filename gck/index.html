1 Transforms using Gray Code Kernels

1.1 Preamble

This work describes some implementation details regarding ”The Gray Code Filter Kernels” [1]. The code can be found here.

1.2 Introduction

The Gray Code Kernels (GCK) is a family of kernels that was shown in [1] to be computable with just two operations per sample, per basis. This implementation uses the Walsh-Hadamard (WH) kernels that were derived in the original paper. The 1-D GCK kernels are defined in [1] recursively:

V s(0) = s, (1)
V s(k) = {[v s(k−1)α kvs(k−1)]}st.v s(k−1) ∈ V s(k−1),α k ∈{+1,-1} (2)

where αkv indicates the multiplication of the vector v by αk and [...] indicates concatenation. For our purposes, s = [1] which generates the WH basis set in V .

Now, the sequence α = (α1...αk) uniquely defines a kernel v ∈ V s(k), referred to as the α-index. Two kernels v+,v− are α-related iff the hamming distance of their α-indices is one. Hence the two kernels have the form (α1...αr−1,+1,...,αk) and (α1...αr−1,−1,...αk). Define their sum vp and difference vm:

vp = v+ + v− (3)
vm = v+ − v− (4)
... (5)
[0Δvp] = [vm0Δ] (6)
vp(i − Δ) = vm(i) (7)
... (8)
v+(i) = +v+(i − Δ) + v−(i) + v−(i − Δ) (9)
v−(i) = −v−(i − Δ) + v+(i) − v+(i − Δ) (10)
... (11)
b+(i) = +b+(i − Δ) + b−(i) + b−(i − Δ) (12)
b−(i) = −b−(i − Δ) + b+(i) − b+(i − Δ) (13)

where Δ = 2r−1|s| is the length of the common prefix vector between v+ and v−, |s| the length of s and 0Δ a zero-vector of length Δ. b0 denotes the first result of convolving an input signal with v0; from there, subsequent results b1,b2...b|s| can be calculated using only previous results. For a complete derivation of all these equations, see [1].

1.3 α-Terms

The definitions from Section 1.2 lead to interesting properties that can be used to our advantage. First is the observation that the α-index uniquely defines a kernel, and it is the α-relation that determines the GCK filtering procedure. Because of that, there is no need to explicitly generate the basis set V for WH kernels. To exploit this, construct the binary representation of α, where + maps to 0, and - to 1:

The key relationship in Table ?? is that α-decimal is simply the Gray code representation of the index value. From there, the GCK computation of the relevant α-terms can proceed with simple bit-twiddling operations as found in Section 1.5.

1.4 b-Terms

Recall the term bp(i − Δ) = bm(i) from Section 1.2. In the signals sense, the system y(t) = x(t−τ) indicates a time delay – that is, the output at t = τ is the input at t = 0 [2]. In practice, that means the current position t = i− Δ gets its delayed result after Δ iterations. To avoid a dependency on the future, instead of delaying the signal, advance it:

bp(i − Δ) = bm(i) (14)
bp(i) = bm(i + Δ) (15)
b+(i) + b−(i) = b+(i + Δ) − b−(i + Δ) (16)
b+(i) = b+(i + Δ) − b−(i + Δ) − b−(i) (17)
b−(i) = b+(i + Δ) − b−(i + Δ) − b+(i) (18)

This notation is a bit backwards in terms of computer code – but it allows us to use W-H results, previously computed Δ iterations ago, in just two subtractions per pixel, per base.

1.5 Implementation

From our derivation of the α-terms and the b-terms, the implementation becomes clear. The 1-D case is straightforward as shown in Algorithm 1 which computes b(0), b(1), ..., b(w) in sequency order for an output of length w, following Table 1.

int* gck1d(int *data, int data_len, int kern_len, int bases) 
{ 
    int i, j, w = data_len + kern_len - 1; 
    int res[len*bases], *cur = res, *prev; 
    int bits = log2(kern_len) - 1; 
 
    calculate_dc(data, cur, data_len, kern_len); // first kernel 
    for (int i = 1; i < bases; i++) { 
        int alpha = gray_code(i); 
        int prefix = prefix_len(gray_code(i-1), alpha, bits); 
        int delta = 1 << prefix; 
        int sign = (alpha >> (bits - prefix)) & 1; // determine + or - 
        prev = cur; 
        cur += len; 
        for (int j = 0; j < delta; j++) cur[j] = -prev[j]; // zero-pad j < delta 
        for (; j < w; j++) { 
            if (sign) cur[j] = cur[j - delta] - cur[j - delta] - prev[j]; 
            else cur[j] = cur[j - delta] - prev[j - delta] - prev[j]; 
        } 
    } 
    return res; 
}
Algorithm 1: 1-D GCK

The 2-D case takes a bit more thought. First, an α-related ordering must be defined for 2-D kernels; we produce a path similar to Figure 2. Adjacent kernels (whether from top or left) are α-related, and are re-used (with a preference for kernels from the left). When re-using a result from the left, the GCK computation is run against the α-related result from the top (and vice versa). In this way, each kernel result can be computed with a single pass through the image.

The horizontal case is simply the 1-D transform, where b(0), b(1), ..., b(w) is computed, and repeated for each row of length w. The vertical case attempts to operate on contiguous blocks as much as possible to preserve locality, so the work is transposed – b(i) is processed in one scan for each element of row i before moving on to b(i + 1) at the next row, and so on.

1.6 Supplemental Algorithms

int gray_code(int a) 
{ 
    return (a >> 1) ^ a; 
}
Algorithm 2: Decimal to Gray Code conversion
int prefix_len(int a, int b, int bits) 
{ 
    int count = 0, mask = 1 << bits; 
    while (mask && (a & mask) == (b & mask)) { 
        count += 1; 
        mask >>= 1; 
    } 
    return count; 
}
